Internet Protocols

Link Layer

  • Responsible for the physical transmission of data across media.
  • Handles packet creation, transmission, reception, error detection, and collisions.

Internet Layer

  • Provides a the best way of connection.
  • Makes use of IP addresses.

Transport Layer

  • Ensures transmissions arrive in order and without error.

Application Layer

  • Protocols that allow applications to interact with Transport Layer.
  • Some protocols include:
    • HTTP: web communication
    • SSH: secure shell connect to server
    • FTP: File transferring
    • POP/SMTP: Email transfer protocols

DNS and URL

Hostname System

  • Developed in the early days of TCP/IP.
  • Hostnames are alphanumeric names assigned to each computer in the system.
  • If an operating system encounters a hostname it consults a host file to find the respective IP address.
  • Works well on smaller systems, but becomes inefficient on larger networks.
  • Multiple computers cannot access the hostfile at the same time.

Domain Name System (DNS) and URL

  • DNS divides a namespace into hierarchical entities called domains.
  • A fully qualified domain name (FQDN) includes the domain name with the hostname.
  • URL stands for Uniform Resource Locators
  • Example: http://www.reddit.com/index.php?page=17#article
    • "http://" : protocol
    • "www" : 3rd level
    • "reddit" : 2nd level
    • "com" : top-level domain
    • "/index.php" : path
    • "?page=17" : query string
    • "#article" : fragment

HTTP and Browsers

  • HTML: hypertext markup language
  • HTTP: hypertext transfer protocol
  • HTTPS uses TLS to secure connection.
  • Both HTML and HTTP were developed at the same time by the same person.
  • Connectionless protocols disconnect from the server after making a request and when the response is ready the connection is re-established.
  • In a stateless protocol the client and server know about each other only during the request. Once the connection is closed they must reconnect as if it was the first time.
  • A request header includes data about client. A response header includes data of server.
  • Client request codes:
    • GET : retrieves the representation of pages
    • HEAD : same as GET, except without content
    • POST : delivers information to server
