System Design

Object Oriented Design

  • Encapsulation
    • The grouping of related ideas into one unit and then can be referenced by a single name.
    • Encapsulation allows abstracting which hides things away from the user when they do not need to know how it operates.
  • Inheritance
    • Inheritance occurs when subclasses are derived from parent classes and act as refinements of the parent.
    • This is done so code does not become repetitive.
  • Polymorphism
    • The ability for an object to take on many forms.
    • Most common use is when a parent class references a child class object.

Testing

Testing vs Debugging

  • Testing is the means of determining if a bug exists.
  • Debugging is the process of trying to find the bug in the code.

Types of Errors

  • Syntax Errors
    • Occurs when a typo exists.
    • Easiest to find since they are picked up during compilation.
  • Run-time Errors
    • Program crashes during execution.
    • An example of this is a divides by zero error.
  • Logic Errors
    • Hardest errors to pick up.
    • Program does not crash, but instead gives the wrong output.
    • Results from incorrect thought processes.

Types of Testing

  • Unit Testing: Also known as component testing, verifies functionality of a secific section of code.
  • Integration Testing: Seeks to verify the interfaces betwen components (modules) against a software design.
  • Component Interface Testing: Checks handling of data between system components.
  • System Testing: Tests a completely integrated system.
  • Acceptance Testing: Sponsor validates the system is up to par and is what was asked.

Testing approaches

  • Black-box testing
    • - Testing without knowledge on how the method works
    • - Just tests inputs/outputs
  • White-box testing
    • Tests every line of code in program
    • Checks internal workings of an application as opposed to functionality
