Project Management

Projects

What is a project?

  • A project is unique, temporary, requires resources, and has uncertainty and stakeholders.
  • Most effort is put forth during the middle of the projects life.
  • Most uncertainty exists at the end of the project.

Project Lifecycle

  • The most basic model for a project is the waterfall model.
  • In the waterfall model you only work forwards and do not go backwards in stages.
  • In CS the waterfall model is typically represented by:
    • Inception
    • Requirements
    • Analysis
    • Design
    • Implementation
    • Testing/Debugging
    • Production
    • Distribution
    • Maintenance
  • The Project Manager is responsible for organizing these steps properly and planning the schedule.

Project Managers

The Vital Dozen

  • Understand the context of project management.
  • Recognize project team conflict as progress.
  • Understand who the stakeholders are and what they want.
  • Accept and use the political nature of organizations.
  • Lead from the front.
  • Understand what "success" means.
  • Build and maintain a cohesive team.
  • Enthusiasm and despair are both infectious.
  • One look forward is worth two looks back.
  • Remember what you are trying to do.
  • Use time carefully or it will use you.
  • Above all, plan, plan, plan.

The Waterfall Model

Phases

  • Requirement Gathering and Analysis:
    • All the requirements of the project are captured in this phase.
    • The requirements are documented in a requirement specification doc.
    • The requirements doc must address these four key points:
      • - Discription of the problem
      • - Basic approach of solution
      • - Constraints (time, budget, etc)
      • - How success will be measured
  • System Design
    • The requirements from the previous phase are studied in this phase and designing begins.
    • Helps to specify hardware that may need to be required and helps define the overall architecture.
  • Implementation
    • Using inputs from the system design, small programs called units are developed.
    • Each unit is tested for functionality and are integrated into the next phase.
  • Integration and Testing
    • All the units from the previous phase are integrated into a system.
    • The created system is then tested for faults and errors.
  • Deployment of System
    • Once all testing and debugging is completed, the system can then be deployed to customers or the market.
  • Maintenance
    • Issues may arrise once the system is deployed to the customer environment.
    • This phase releases patches and updates to these issues.
